nginx+tomcat session共享踩坑记

发表于 2017-08-29 | 浏览 次

0x00 环境

我司南京建邺一项目采用nginx+tomcat共享session实现负载均衡方案，配置方案：

nginx+tomcat*2+tomcat-redis-session-manager

0x01 问题

用户登录偶然性登录失败

0x02 分析

首先，通过浏览器debug定位到登录失败原因在于有个接口请求报了500错误
http://www.xxxx.gov.cn/menu/queryMenusByUser

通过代码可以看出，如果报错会有stacktrace，但是浏览器中的Response并没有输出，于是查看服务器日志，频繁报出如下错误:

意思是在反序列化的过程中找不到类。但是这个类是shiro-core中的一个类，而在服务器项目的lib下面也是可以找到的。

查看tomcat的server.xml中发现：

项目中配置了2个context，而且这两个context属于包含与被包含的关系，通过分析tomcat的valve机制，找到问题原因在于redis中取出session进行反序列化操作的时候反序列化到/uploadfile这个context中，而这个/uploadfile中是没有SimplePrincipalCollection这个类的，所以报了java.lang.ClassNotFoundException。

0x03 解决

通过分析，解决方案就是进行架构调整，不再使用tomcat作为附件服务容器，而是通过nginx来提供给附件服务，这样做不仅可以避免反序列化问题，而且还可以利用nginx的缓存加速附件访问。

# nginx # tomcat # session-manager
