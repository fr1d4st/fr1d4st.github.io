CentOS启用Tomcat APR+Https

发表于 2017-09-07 | 浏览 次

0x00 背景

  最近我司一项目tomcat需要全站Https，客户提供了2个文件：

  • xxx.crt
  • xxx.key

  系统运行环境：

  • CentOS 6.x
  • Tomcat 7.x
  • JAVA7

0x01 tomcat运行模式

  对于一般个人用户来说，tomcat启用Https可以通过jvm提供的keytool工具来生成.keystore文件，然后在tomcat中配置keystoreFile和keystorePass，keytool位于/path/to/JDK/jre/lib/security/目录。但是自己生成的证书文件，浏览器是不认可的。所以，自己生产证书文件并配置的Https在浏览器中会显示站点连接不可信，并有红色反斜杠标志，没有我们希望得到的保护色的漂亮Https标志。

  由于我们客户是提供了证书文件，而且用户希望看到保护色的漂亮标志，因此，我们必须利用客户提供的文件来启用全站Https。

  tomcat有3种运行模式，如下：

运行模式 protocol 说明
BIO org.apache.coyote.http11.Http11Protocol 阻塞式I/O，tomcat8一下默认启用模式。
NIO org.apache.coyote.http11.Http11NioProtocol 非阻塞式I/O，通过JDK提供的java.nio.*实现，tomcat8默认启用模式。
APR org.apache.coyote.http11.Http11AprProtocol 非阻塞式I/O，通过系统级类库APR实现，性能最高，默认不启用。

APR：Apache Portable Run-time libraries，Apache可移植运行库
在早期的Apache版本中，应用程序本身必须能够处理各种具体操作系统平台的细节，并针对不同的平台调用不同的处理函数。随着Apache的进一步开发，Apache组织决定将这些通用的函数独立出来并发展成为一个新的项目。这样，APR的开发就从Apache中独立出来，Apache仅仅是使用APR而已。

  在这三种模式中，Https的配置还不一样，BIO和NIO模式使用keystoreFile和keystorePass来配置，而APR模式中使用SSLCertificateFile和SSLCertificateKeyFile来配置。由于客户提供给我们的是.crt文件，而不是.keystore文件，因此我们只能使用APR模式。

Tomcat Native：这个项目可以让 Tomcat 使用 Apache 的 apr 包来处理包括文件和网络IO操作，以提升性能。

官网介绍：
The Apache Tomcat Native Library is an optional component for use with Apache Tomcat that allows
Tomcat to use certain native resources for performance, compatibility, etc.
（大概意思是Tomcat可以利用一些native资源来提高性能和兼容性。）

Specifically, the Apache Tomcat Native Library gives Tomcat access to the Apache Portable Runtime
(APR) library’s network connection (socket) implementation and random-number generator.
（具体来说是利用了APR库中的网络连接实现和随机数生成器。）
