编译安装nginx+sticky

发表于 2017-04-28 | 浏览 次

0x00 关于

sticky模块（nginx-sticky-module）
普通安装的nginx是不带sticky模块的，安装sticky模块需要重新编译nginx。
没有sticky模块的nginx主要通过ip_hash把同一来源的客户(同一C段的IP)固定指向后端的同一台机器，这样并不能实现很好的负载均衡，（针对同一网段的用户）。
增加了nginx-sticky-module的nginx，首先根据轮询RR随机到某台后端，然后在响应的Set-Cookie上加上route=md5(upstream)字段，第二次请求再处理的时候，发现有route字段，直接导向原来的那个节点。
如下:
route=e6b57a7d2c523d67adf934eb8213f4c8; JSESSIONID=ABE2BF19945483F6EB73DB7591F4838D

0x01 配置说明

  • name: 可以为任何的string字符,默认是route
  • domain：哪些域名下可以使用这个cookie
  • path：哪些路径对启用sticky,例如path/test,那么只有test这个目录才会使用sticky做负载均衡
  • expires：cookie过期时间，默认浏览器关闭就过期，也就是会话方式。
  • no_fallbackup：如果设置了这个，cookie对应的服务器宕机了，那么将会返回502（bad gateway 或者 proxy error），建议不启用。
