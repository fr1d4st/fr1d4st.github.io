生产环境JVM调优

发表于 2018-01-24 | 浏览 次

RHEL7
JAVA8
16G

-Xms -Xmx

JVM最小内存和JVM最大内存

-Xss

每个线程的堆栈大小

-XX:+AggressiveOpts

每当JDK版本升级时，你的JVM都会使用最新加入的优化技术

-XX:+UseBiasedLocking

启用优化线程锁

-XX:PermSize -XX:MaxPermSize

-XX:PermSize 非堆内存初始值，默认是物理内存的1/64
-XX:MaxPermSize 最大非堆内存的大小，默认是物理内存的1/4

-XX:+DisableExplicitGC

禁止代码中的显式gcSystem.gc()

-XX:+UseParNewGC

对年轻代采用多线程并行回收

-XX:+UseConcMarkSweepGC

使用CMS gc

-XX:MaxTenuringThreshold

设置垃圾最大年龄

-XX:+CMSParallelRemarkEnabled

在使用UseParNewGC 的情况下, 尽量减少 mark 的时间

-XX:+UseCMSCompactAtFullCollection

在使用concurrent gc的情况下, 防止memoryfragmention, 对live object进行整理,使memory碎片减少

-XX:LargePageSizeInBytes

Java heap的分页页面大小

-XX:+UseFastAccessorMethods

get,set 方法转成本地代码

-XX:+UseCMSInitiatingOccupancyOnly

指示只有在 oldgeneration 在使用了初始化的比例后concurrent collector 启动收集

-Djava.security.egd=file:/dev/./urandom

优化熵源
